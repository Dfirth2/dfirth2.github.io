Why This Artifact Was Selected

I selected the Thermostat Project because it gave me a practical foundation for showing how software can control real-world hardware. The original project used embedded system logic to monitor temperature, control heating and cooling modes, and respond to physical button input on a Raspberry Pi. Because it already connected software decisions to physical behavior, it was a strong artifact to expand for my capstone.

Enhancing this project allowed me to show growth beyond the original assignment by turning a local-only thermostat into a full-stack IoT-style system. I improved the software design with a web dashboard and backend API, added scheduling logic to manage thermostat events, and included database storage so settings, schedules, and activity could persist over time. These improvements make the project a strong representation of my progress in software engineering, algorithms and data structures, databases, and security.

Enhancement Categories

Implemented Growth Areas

01

Software Design and Engineering

The enhanced version expands the original Raspberry Pi thermostat into a full-stack IoT-style system. The project now separates responsibilities between the Angular dashboard, FastAPI backend, SQLite database, authentication controls, and Raspberry Pi device logic. This improves maintainability, testing, security, and future expansion.

02

Algorithms and Data Structures

The enhancement adds scheduling logic that uses structured event objects and collections to manage thermostat events. Events are sorted by day and time, filtered by user and status, checked for conflicts, and processed to identify the next scheduled event. This allows the thermostat to follow planned behavior instead of relying only on manual controls.

Narrative ⤓
03

Databases

The database enhancement adds persistent storage for users, thermostat settings, scheduled events, and activity logs. The system uses backend API routes to create, retrieve, update, and manage stored data, allowing the dashboard and Raspberry Pi thermostat to share consistent information across the full application.
