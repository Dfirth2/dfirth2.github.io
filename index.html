Self-Assessment

Dayne Firth II

My capstone portfolio connects practical technical experience with the computer science skills I developed through the program.

Professional Focus

Building reliable, secure, and user-focused software that solves practical problems.

My professional background has always been centered on solving real problems. Before beginning my computer science journey, I worked in hands-on technical roles as an installer, lineman, and outside plant design engineer for AT&T. These experiences taught me how important it is for technology to be practical, reliable, and useful to the people who depend on it every day.

From Field Experience to Software Development

Starting the Computer Science program felt like the next step in that journey. I was no longer only solving problems in the field; I was learning how to build the tools and systems that could solve those problems through software. Each course added a new skill to that path, from communication and planning to algorithms, databases, software engineering, and security.

As I moved through the program, I began to see how these different areas connected to form complete solutions. This experience helped me grow from someone who wanted to build useful tools into someone who better understands how to design, develop, test, and secure practical software applications.

Communication and Collaboration

An important part of the journey has been learning how to communicate and collaborate. In my professional work, I have communicated with field technicians, engineers, and leadership to make sure projects are understood and completed correctly. From that experience, I learned that technical knowledge alone is not enough.

A strong solution also depends on understanding user needs, explaining decisions clearly, and working with others to solve problems. Throughout the Computer Science program, I continued developing this skill through written documentation, project narratives, code reviews, discussions, and visual aids.

Technical Growth Across the Program

The technical part of my journey, where I had the most fun, helped me understand how different areas of computer science work together to create complete solutions. In software engineering and design, I developed the ability to plan applications around user needs, organize code into manageable parts, and test systems to make sure they work as intended.

Through data structures and algorithms, I came to appreciate how important it is to organize information efficiently and apply logic to solve problems, such as sorting data, searching records, and handling scheduling decisions. My database coursework gave me insight into how applications store and manage information through tables, relationships, and persistent data. Security also became a major part of my development process because every system must protect users, validate input, and reduce risks before problems occur.

Capstone Artifact and Career Direction

The artifacts in my portfolio show how I applied these skills to improve an existing project into a more complete and practical solution. The main artifact I selected was my thermostat project, which began as an embedded systems assignment using a Raspberry Pi to monitor temperature and control heating and cooling behavior.

Through the enhancement process, I expanded the project beyond a local device and turned it into a more complete software system. The enhanced version includes a frontend dashboard, backend API, database support, scheduling logic, device integration, testing, and security improvements. Each enhancement connects to a different area of computer science, but together they show my ability to analyze an existing system, identify areas for improvement, and build a more useful and secure application.

Overall, my experience in the Computer Science program has helped prepare me to move forward as a computer science professional. I entered the program with real-world technical experience and a desire to build tools that could solve practical problems. This portfolio represents that growth and shows my ability to connect practical experience with computer science knowledge.
